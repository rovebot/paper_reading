1. ObjectBox: From Centers to Boxes for Anchor-Free Object Detection

[info] 基本信息

2022ECCV oral，加拿大皇后大学

MS-COCO, PASCAL VOC, 目标检测，anchor-free, 标签分配，IoU

ISSUES区已被怼烂

[info] 背景

anchor-based的缺点： 首先，使用预定义的锚点会引入额外的超参数来指定它们的大小和纵横比，这会损害对其他数据集的泛化。 其次，锚点必须密集地覆盖图像以最大化召回率。少数锚点与大多数地面真值框重叠，导致正负锚框之间的巨大不平衡，并增加了额外的计算成本，从而减慢了训练和推理 [16]、[3]。——topk类的标签分配都解决了这个问题 第三，必须仔细设计锚框的数量、比例和纵横比，因为改变这些参数会影响性能。

anchor-free: 在基于关键点的方法中，使用标准关键点估计网络（例如 HourglassNet [21]）定位多个对象点，例如中心点和角点，并将其分组以限制对象的空间范围。然而，它们在关键点检测之后需要复杂的组合分组算法。相比之下，基于中心的方法更类似于基于锚点的方法，因为它们使用感兴趣的对象区域或中心位置来定义正样本。基于锚点的方法根据锚框的中心位置预定义的锚框，anchor-free 方法直接回归这些位置的边界框。

基于中心的方法中基于锚点的方法和无锚点方法之间的主要区别在于正训练样本和负训练样本的定义，这导致了性能差距。为了区分正样本和负样本，anchor-based 方法使用 IoU 同时在空间和尺度维度上选择正样本，而 anchor-free 方法使用一些空间和尺度约束首先在空间维度上找到候选正样本，然后在尺度维度。然而，两种静态策略都施加约束阈值来确定正样本和负样本之间的边界，而忽略了对于具有不同大小、形状或遮挡条件的对象，最佳边界可能会有所不同的事实[6]。为了应对这个问题，已经开发了许多动态分配机制 [37]、[6]、[13]。例如，在[37]中，建议根据一些统计标准为每个目标设置划分边界。

[info] 贡献

提出了一个新的尺度不变标准作为 IoU 度量，提出了一种新颖的无锚对象检测器 ObjectBox，它可以更好地处理标签分配问题

[info] 方法

ObjectBox和FCOS搞不懂有啥区别。只不过一个以特征点对应中心像素点作为原点计算偏移，一个以特征点对应的像素单元作为原点计算偏移

2. RFLA: Gaussian Receptive Field based Label Assignment for Tiny Object Detection（微小物体的高斯分布标签分配）

同 Detecting tiny objects in aerial images: A normalized Wasserstein distance and a new benchmark

[info] 背景

微小的物体只包含几个像素，由于缺乏对判别特征的监督，最先进的物体检测器在微小物体上不能提供令人满意的结果。

[info] 痛点

交并比 (IoU) 度量及其扩展对微小对象的位置偏差非常敏感，IoU对不同尺度物体的敏感性差异很大，当用于基于锚的检测器时，这会极大地降低标签分配的质量。

[info] 两大现象

  1. IoU 对不同尺度物体的敏感性差异很大，特别是对于微小物体的位置偏差更敏感。导致标签分配不准确。IoU的敏感性源自于边界框位置只能离散变化的特殊性,换句话说，随着物体尺度的增加，IoU的值从单位像素偏差的变化逐渐接近于连续变化，因此在检测正常大小的物体时通常会忽略IoU的离散性。——但是预测结果都是连续的，因此计算IoU也是连续的。

  2. 第二个观察是，当两个边界框没有重叠或相互包含时（即 IoU 的值保持不变），IoU 无法反映两个边界框的位置关系，这通常是微小边界框的情况。——改进的IoU可以，例如DIoU等。

[info] 导致的三大问题

  1. IoU 对微小物体的敏感度使得大多数anchor候选都因为微小的位置偏差变成了负样本。——OTA，SimOTA都解决了这个问题

  2. 基于 IoU 的分配存在严重的规模不平衡问题。其中较大尺度的对象往往比微小对象分配更多的 pos 样本。因此，网络倾向于优化更大规模的对象。虽然 ATSS (Zhang et al., 2020) 等动态分配策略可以根据对象的统计特征自适应地设置 IoU 阈值来分配 pos/neg 标签，但 IoU 的敏感性使得很难找到一个好的阈值并提供高质量的用于微小物体检测器的 pos/neg 样本。——但是SimOTA等分配方法求TOP k在一定程度上解决了这个问题

  3. 最后，样本补偿分配策略在当前的测量方式下效果不佳。为了使每个实例得到充分训练，通常会部署启发式算法以保证每个 gt 至少有一个训练样本（即使没重叠）（Ren et al., 2015; Zhang et al., 2017; Xu et al., 2021），我们称这种方法为样本补偿策略。然而，当两个盒子分开时，IoU 保持为零。在这种情况下，很难根据相同的 IoU 分数提供合理的 anchor 候选排名。 ——完全不重叠被强制分配为正样本不合理

[info] 方法

作者提出了一种新的评估指标，提出了一种新的度量来衡量微小边界框的相似性，并将其与定制的标签分配策略相结合。称为归一化 Wasserstein 距离 (NWD) 和一种新的基于 RanKing 的分配 (RKA) 策略，用于微小物体检测。

首先将边界框建模为二维高斯分布，然后使用经典的 Wasserstein 距离来测量派生的高斯分布的相似性。

然后选择指数非线性变换函数来重新映射高斯 Wasserstein 距离并对其取值范围进行归一化，我们将得到的度量标准称为归一化 Wasserstein 距离（NWD）。

[info] NWD

基于 Wasserstein 距离设计了一个更好的针对微小对象的度量，因为它可以一致地反映分布之间的距离，即使它们没有重叠。因此，新指标在测量微小物体之间的相似性方面比 IoU 具有更好的特性。——没有重叠就不应该被分配为正样本

为了获得类似于 IoU 的值范围（即在 0 和 1 之间）并使度量平滑到位置偏差，我们启发式地选择指数非线性变换函数将高斯 Wasserstein 距离重新映射到另一个空间，从而将其值范围归一化为（ 0, 1]。

[info] # assignment

Faster RCNN等基于阈值的分配方法对于不同尺度物体的锚点分配极不平衡，因此对微小物体的监督不足。为了缓解这个问题，我们提出了一种新的基于排名的分配策略，并将其与 NWD 相结合，以充分利用新指标在微小对象上的优势。——基于排名top k的方法在最近两年的标签分配工作里都有了，不应该只和FRCNN比较

[info] 作用

对于微小的物体，通常它们的边界框与几乎所有的锚点都没有重叠。 Wasserstein 距离的主要优点是即使分布的重叠可以忽略不计，它也可以测量分布的相似性。因此，所提出的 NWD 可以一致地反映 IoU 度量无法实现的非重叠微小对象之间的位置关系。（DIOU一类的就可以）

此外，NWD 度量对位置偏差具有平滑性，具有尺度平衡的特点。如图 2 所示，不同尺度的 NWD-Deviation 曲线重合，这意味着直接时可以将更多的正样本分配给微小的物体。（会分配大量低质量的正样本）
